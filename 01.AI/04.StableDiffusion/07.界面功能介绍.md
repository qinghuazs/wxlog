## 1. txt2img（文生图）

文生图是 WebUI 最核心的功能，通过输入文字描述生成图像。

```mermaid
    style B fill:#cfe2ff,stroke:#333
    style C fill:#d4edda,stroke:#333
```

### 1.2 提示词区域

#### Prompt（正面提示词）

```
作用：描述你想要生成的图像内容
位置：界面顶部大文本框

示例：
masterpiece, best quality, 1girl, long hair, blue eyes,
white dress, standing in flower field, sunlight, detailed

技巧：
- 重要内容放前面
- 用逗号分隔不同元素
- 可使用权重语法 (keyword:1.2)
```

#### Negative Prompt（负面提示词）

```
作用：描述你不想出现的内容
位置：Prompt 下方的文本框

示例：
low quality, worst quality, blurry, watermark, text,
bad anatomy, bad hands, missing fingers, extra limbs

技巧：
- 添加常见质量问题
- 使用负面 Embedding（如 EasyNegative）
- 针对性添加（如人物图加 bad hands）
```

```mermaid

```

#### Sampling Method（采样方法）

| 方法 | 特点 | 推荐场景 | 速度 |
|------|------|----------|------|
| **Euler a** | 创意性强，变化大 | 探索阶段 | 快 |
| **Euler** | 稳定，收敛快 | 固定种子复现 | 快 |
| **DPM++ 2M Karras** | 质量高，细节好 | 日常使用首选 | 中 |
| **DPM++ SDE Karras** | 细节丰富，略慢 | 追求质量 | 慢 |
| **DPM++ 2M SDE** | 均衡之选 | 通用场景 | 中 |
| **DDIM** | 确定性强 | 需要精确控制 | 中 |
| **UniPC** | 高效率 | 快速出图 | 快 |

#### Sampling Steps（采样步数）

```
步数与质量关系：
├── 10-15 步：草图质量，快速预览
├── 20-30 步：标准质量，日常使用（推荐）
├── 40-50 步：高质量，精细图像
└── 50+ 步：边际效益递减，不推荐

注意：不同采样器对步数敏感度不同
- Euler a：20 步即可
- DPM++ 2M Karras：30 步较佳
```

### 1.5 CFG Scale（提示词相关性）

```
CFG Scale 控制图像与提示词的匹配程度

数值范围：1-30（常用 5-15）

效果对比：
├── 1-4：创意自由，可能偏离提示词
├── 5-7：轻度引导，保持创意
├── 7-9：标准范围，推荐使用
├── 10-12：强引导，严格遵循提示词
├── 13-15：非常严格，可能过饱和
└── 15+：通常过度，画面异常

推荐值：7（新手默认）
```

#### Seed 使用技巧

```
1. 探索阶段：使用 -1（随机）
2. 找到喜欢的图：记录 Seed 值
3. 微调优化：固定 Seed，调整其他参数
4. 批量变体：使用 Variation seed + Variation strength

Extra 选项：
- Variation seed：变体种子
- Variation strength：变体强度（0-1）
- Resize seed：种子尺寸调整
```

### 1.7 批量生成

```
Batch count：生成批次数（顺序执行）
Batch size：每批生成数量（并行执行）

示例：
Batch count = 4, Batch size = 2
→ 共生成 4 × 2 = 8 张图
→ 每次并行生成 2 张，执行 4 次

显存建议：
- 6GB 显存：Batch size ≤ 2
- 8GB 显存：Batch size ≤ 4
- 12GB+ 显存：Batch size ≤ 8
```

```mermaid
    style D fill:#d4edda,stroke:#333
    style E fill:#d4edda,stroke:#333
```

### 2.2 核心参数：Denoising Strength

```
Denoising Strength（重绘强度）
范围：0-1

效果对比：
├── 0.1-0.3：轻微修改，保持原图结构
├── 0.4-0.5：中度修改，风格转换
├── 0.6-0.7：大幅修改，保留构图
├── 0.8-0.9：几乎重绘，仅参考构图
└── 1.0：完全重绘，忽略原图

常用场景：
- 照片转动漫：0.5-0.7
- 优化细节：0.2-0.4
- 风格迁移：0.6-0.8
- 草图上色：0.7-0.9
```

### 2.3 Inpaint（局部重绘）

```mermaid
graph LR
    A[Inpaint 流程] --> B[上传图片]
    B --> C[绘制蒙版]
    C --> D[输入提示词]
    D --> E[设置参数]
    E --> F[生成结果]

    subgraph 蒙版设置
        C --> C1[黑色保留]
        C --> C2[白色重绘]
    end

    subgraph 关键参数
        E --> E1[Mask blur 边缘模糊]
        E --> E2[Inpaint area 重绘区域]
        E --> E3[Denoising 重绘强度]
    end

    style C fill:#cfe2ff,stroke:#333
    style E fill:#d4edda,stroke:#333
```

#### Inpaint 参数详解

| 参数 | 作用 | 推荐值 |
|------|------|--------|
| **Mask blur** | 蒙版边缘模糊度 | 4-8 |
| **Mask mode** | Inpaint masked / not masked | 根据需求 |
| **Masked content** | 蒙版区域填充方式 | Original 或 Latent noise |
| **Inpaint area** | Whole picture / Only masked | Only masked（节省显存） |
| **Only masked padding** | 边缘填充像素 | 32-64 |

#### Masked Content 选项

```
fill：纯色填充，适合大面积重绘
original：保持原图，适合微调
latent noise：噪声填充，创意性强
latent nothing：潜空间空白，中性起点
```

### 2.4 Sketch（草图上色）

```
使用流程：
1. 在画布上绘制草图（或上传草图）
2. 输入描述提示词
3. 设置较高的 Denoising（0.7-0.9）
4. 生成上色结果

技巧：
- 简单线条即可，不需精细
- 提示词描述颜色和风格
- 多次生成选择最佳
```

### 3.2 图像放大

#### 放大器对比

| 放大器 | 特点 | 适用场景 | 速度 |
|--------|------|----------|------|
| **Lanczos** | 锐利，保持边缘 | 像素画、线稿 | 快 |
| **ESRGAN_4x** | 通用放大 | 一般图像 | 中 |
| **4x-UltraSharp** | 高清锐利 | 写实照片 | 中 |

| **R-ESRGAN 4x+** | 平衡之选 | 通用 | 中 |
| **R-ESRGAN 4x+ Anime6B** | 动漫专用 | 二次元 | 中 |
| **SwinIR_4x** | 高质量 | 追求极致 | 慢 |
| **LDSR** | 最高质量 | 最终输出 | 很慢 |

#### 推荐设置

```
写实图像放大：
- Upscaler 1：R-ESRGAN 4x+
- Upscaler 2：4x-UltraSharp
- Upscaler 2 visibility：0.3-0.5

二次元图像放大：
- Upscaler 1：R-ESRGAN 4x+ Anime6B
- Upscaler 2：4x-AnimeSharp
- Upscaler 2 visibility：0.3-0.5
```

### 5.2 重要设置项

#### Saving images/grids（图像保存）

```
Images filename pattern：文件名格式
推荐：[date]-[seed]-[model_name]

Save images to a subdirectory：按日期分文件夹
推荐：开启

File format for images：保存格式
推荐：PNG（无损）或 JPEG（节省空间）
```

#### Stable Diffusion

```
SD VAE：VAE 选择
- Automatic：自动选择
- None：使用模型内置
- 指定 VAE 文件

Clip skip：跳过 CLIP 层数
- 1：写实模型
- 2：二次元模型（常用）

Enable emphasis：启用权重语法 (keyword:1.2)
推荐：开启

Comma padding backtrack：逗号填充
推荐：20
```

#### Optimizations（优化）

```
Cross attention optimization：注意力优化
推荐：xFormers（需安装）或 sdp

Token merging ratio：Token 合并
- 0：不合并（质量最好）
- 0.3-0.5：适度加速
- 0.7+：明显加速，质量下降

Pad prompt/negative prompt：提示词填充
推荐：开启（减少偏差）
```

#### User interface（用户界面）

```
Quicksettings list：快捷设置项
推荐：sd_model_checkpoint, sd_vae, CLIP_stop_at_last_layers

Show progress in window title：窗口标题显示进度
推荐：开启

Live preview method：实时预览方法
推荐：Approx NN（快速）或 Full（精确）
```

### 5.3 推荐配置

```
新手推荐设置：
├── SD VAE: Automatic
├── Clip skip: 2
├── Cross attention: xFormers
├── Live preview: Approx NN
├── Save format: PNG
└── 其他保持默认

进阶优化设置：
├── Token merging: 0.3（加速）
```


### 7.3 权重语法详解

```
增强权重：
(keyword) = (keyword:1.1)
((keyword)) = (keyword:1.21)
(((keyword))) = (keyword:1.331)
(keyword:1.5) = 直接指定 1.5 倍权重

降低权重：
[keyword] = (keyword:0.9)
[[keyword]] = (keyword:0.81)
(keyword:0.5) = 直接指定 0.5 倍权重

混合语法：
[keyword1:keyword2:0.5] = 前 50% 用 keyword1，后 50% 用 keyword2
[keyword1|keyword2] = 交替使用两个关键词

示例：
(masterpiece:1.2), best quality, ((detailed eyes)), [simple background]
```

## 常见问题

### Q1: 界面加载很慢

```
解决方案：
1. 添加启动参数 --skip-torch-cuda-test
2. 使用 --ui-settings-file 指定配置文件
3. 清理浏览器缓存
4. 检查扩展是否过多
```

### Q2: 生成后图片不显示

```
解决方案：
1. 检查保存路径是否有写权限
2. 查看 outputs 文件夹是否存在
3. Settings → Saving → 检查保存设置
4. 重启 WebUI
```

### Q3: 参数改动不生效

```
解决方案：
1. 点击 Apply settings 应用设置
2. 某些设置需要重启 WebUI
3. 检查是否有扩展冲突
```

## 总结

### 界面快速索引

| 功能 | 标签页 | 主要用途 |
|------|--------|----------|
| 文字生图 | txt2img | 根据提示词生成图像 |
| 图片生图 | img2img | 基于图像生成/修改 |

### 新手上手顺序

```

3. 安装必备扩展（ControlNet、TagComplete）

5. 掌握 Extras 后期处理

```

最后，界面功能很多，不必一次全部掌握。先从 txt2img 开始，生成几百张图后自然就熟悉了。熟悉基础操作后，强烈建议学习 ControlNet，它能大幅提升创作的可控性。
